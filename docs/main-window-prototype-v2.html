方案 C 优劣

  • 视觉冲击最强 —— 评委一眼能看到"正在做什么 / 完成多少"
  • 拖拽改状态自然，不需要点选菜单
  • 每列独立滚动 + 计数 + 添加按钮，工作流清晰
  • 优先级用左侧色条，扫一眼就能看出 T0/T1/T2 分布
  • 需要宽屏（≥ 960pt），13" 笔记本会挤
  • 用户需要手动拖卡片 → 有学习成本（虽然轻）
  • "已完成"列长期堆积会变臃肿，需要"自动归档 7 天前"机制

方案 D 优劣

  • "今日全景"一屏看完 —— 评委 3 秒内 get 到这是个有数据 / 有陪伴 / 有性格 的 app
  • Hero 大卡片 + 巨大数字最有冲击力 (11,616 那种 26pt 数字直接占眼)
  • 4 个 widget 各管一摊：任务 / 进度 / 趋势 / 人，符合"卡片化"潮流
  • 底部输入条 sticky，永远在视线内，添加事件零摩擦
  • 任务列表是缩略版（4 行），看完整列表需要点 "查看全部"
  • 看着信息多，但实际任务管理 (拖拽 / 排序 / 移动) 操作能力弱
  • 最像"看板"而非"工具" —— 长期使用也许会觉得"花瓶感"

现在四个方案都有了：

A · 三栏经典（v1）—— 专业生产力工具，有 sidebar + 右栏总览

B · 时间线流式（v1）—— 按时间分组，安静治愈

C · 看板 Kanban（本页）—— 三列拖拽工作流，最强视觉冲击

D · Mission Control 仪表盘（本页）—— Hero + 4 widget，"今日全景"

告诉我挑哪个 / 哪几个混搭，我把 ContentViewController 重写。
