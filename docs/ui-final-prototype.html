UI 总规范 + 完整界面 · 重新画

v1.7.27 暴露的问题：
① 详情窗口黑底白字（system color 没强制覆盖）
② 部分按钮跟背景同色（"标记完成"、"关闭"、"删除"看不清）
③ 待办列表卡片样式不一致 / 行错位
④ 超宽屏 Kanban 列空旷
本版核心规则（必须严格遵守，不能再依赖系统色）：
① 每个组件的 背景色 + 前景色 必须显式定义，禁用 .labelColor / .windowBackgroundColor
② 所有按钮 = 4 种明确风格之一（primary / secondary / tertiary / danger），不能写 "默认 NSButton"
③ 文字最低对比度 4.5:1（深棕 #2c2418 vs 白底 = 14:1，OK）
④ 主窗口 max-width: 1280pt 居中，超宽屏不"摊大饼"
DESIGN TOKENS · 必须严格
颜色 / 字体 / 阴影 规范
这是后续 Swift 实现要用的全部 token —— 不再写 .labelColor / .windowBackgroundColor，全部替换成下面定义的硬编码值

BUTTONS · 4 种 + 1 个图标按钮 + chip
按钮规范 — 修「按钮看不清」
每种按钮都必须 显式 background + foreground，不能在 NSButton 上 isBordered=false 然后期望系统给颜色。下面是必须用的 6 种。

① PRIMARY 主操作 · 强对比

背景 var(--accent) #FF8C42 + 白字 + accent shadow · 用于：主要 CTA / Submit / 添加

② SECONDARY 次操作 · 中对比

白底 + 深字 + 1.5px 暖米边框 · 用于：取消 / 关闭 / 次要操作（hover 变橙）

③ TERTIARY 文字按钮

透明底 + 橙字 · 用于：链接 / 弱次要

④ DANGER 危险按钮

红底 + 白字（实心）/ 白底 + 红字 + 红边（ghost） · 用于：删除 / 清空 / 不可逆操作

⑤ ICON 图标按钮 · 工具栏专用

米底 + 深字 · hover 变黄油 / active 变橙 · 用于：工具栏、迷你按钮

⑥ CHIP 标签胶囊（进行中 / 完成 切换）

米底 + 深字 → active 橙底白字 · 用于：tab 切换 / 视图切换
主窗口 · 修「太宽 + 行 bug」
居中限宽 1280pt + 紧凑卡片
超宽屏自动留白，卡片高度从 80pt 降到 40pt，一屏看 10 张。check 圆圈 hover/done 状态明确。
