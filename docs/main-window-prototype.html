主窗口布局原型 · 两个方案 PK

当前主窗口的痛点：

① 输入区一行挤 4 个控件（人名 + 优先级 + 定时 + 添加），看着乱
② 2000 字文本区强制占大半屏，但大多数任务就 1 句话，浪费空间
③ 没有分类导航（按人 / 按优先级 / 按时间），所有任务混在一个表里
④ 没有"今日总览"，看不见自己完成了多少
⑤ 强制深色模式，跟我们 Style B 暖橘调撞色

下面两个方案都用 Style B 重新设计，左右各一个。

PROPOSAL A
三栏经典 · Sidebar + 主区 + 总览
最像专业生产力工具的版本。
左 sidebar 分类，中间任务，右栏看今日进度 + 打字 + 提醒。

方案 A 优劣

  • 分类导航直接，按人 / 按优先级 / 按时间一眼切
  • 右栏总览 = 进度环 + 打字数 + 即将到期，每打开就给一份"今日成绩单"
  • 输入卡折叠：默认一行，需要长文/贴图时才展开
  • 屏幕宽度需求 ≥ 880pt，小笔记本可能挤
  • 右栏 + sidebar 可能"信息过载"，新手前 30s 不知道看哪

方案 B 优劣

  • 单栏聚焦，13" 笔记本完美适配
  • 时间线分组贴合"今天该做什么"的心智，不用切来切去
  • 输入卡用 emoji 工具按钮，一行 hold 住所有控件，紧凑 30%
  • 顶部状态条 = 进度 + 键数 + 心情语，每次开窗都被"陪伴"
  • 没有分人 / 分优先级的快捷视图（要靠"按人 / 按时间" toggle 切）
  • 任务多时（>20）滚动列表会很长

挑哪个？

A 适合：你常常按人 / 按优先级筛选，团队协作场景，希望一眼看到所有维度（推荐评委席给"专业感"）

B 适合：你主要是个人记录 + 当天聚焦，喜欢 Things / Reminders 那种安静的单栏，希望评委说"很治愈"

告诉我 A / B / 或者混搭（比如 B 的状态条 + A 的右栏总览），我把 ContentViewController 全部按那套重写。
